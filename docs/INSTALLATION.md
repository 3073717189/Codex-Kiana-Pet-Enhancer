# Codex 琪亚娜增强桌宠：安装、使用与卸载

本文档是当前公开版的准确操作说明。视频主要用于展示桌宠效果，若视频与本文档不一致，请以本文档为准。

> 视频结尾的卸载画面只展示了“Codex 仍在运行，因此卸载被阻止”的保护提示，并不是完整卸载。正确卸载必须先完全退出 Codex，再重新运行卸载器。

## 一、使用条件

- Windows 10/11 x64
- 微软商店安装的官方 Codex，并且至少正常启动过一次
- 不需要管理员权限
- 不需要安装 Git、Node.js 或 PowerShell 7

本项目只增强桌宠动作，不安装主题，也不会修改 Codex 背景、输入框、图标、`WindowsApps`、`app.asar` 或 `.codex\config.toml`。

## 二、下载与校验

1. 打开 [GitHub Releases](https://github.com/3073717189/Codex-Kiana-Pet-Enhancer/releases/latest)。
2. 下载 `Codex-Kiana-Pet-Enhancer-v1.0.0-win-x64.zip`。
3. 建议同时下载对应的 `.zip.sha256.txt`。
4. 在 ZIP 所在目录打开 PowerShell，运行：

```powershell
Get-FileHash '.\Codex-Kiana-Pet-Enhancer-v1.0.0-win-x64.zip' -Algorithm SHA256
```

输出的哈希应与 `.sha256.txt` 完全一致。

## 三、首次安装

1. 完整解压 ZIP。不要在压缩包预览窗口中直接运行文件。
2. 保存尚未发送的内容，然后完全退出所有 Codex 窗口。
3. 双击解压目录中的 `安装 Codex 琪亚娜增强桌宠.cmd`。
4. 等待出现安装成功提示。安装器只写入桌宠和增强启动器，不会自动选择桌宠，也不会自动启动 Codex。
5. 使用原来的普通 Codex 入口启动一次 Codex。
6. 打开 `设置 > Pets（宠物）`，刷新自定义桌宠并选择“时砾逐光”。
7. 完全退出 Codex。
8. 此后使用桌面或开始菜单中的“Codex 琪亚娜桌宠”启动。

如果安装器提示 Codex 正在运行，本次安装没有继续。请完全退出 Codex 后重新运行安装器。

## 四、以后怎么启动

| 启动入口 | 实际效果 |
| --- | --- |
| “Codex 琪亚娜桌宠” | 加载工作状态识别、自动奔跑、视线跟随和彩蛋等增强逻辑 |
| 原来的普通 Codex | 仍可正常使用，但不会加载本项目的增强逻辑 |

首次选择“时砾逐光”后仍没有增强动作，通常是因为普通 Codex 尚未完全退出。关闭所有 Codex 窗口，再使用“Codex 琪亚娜桌宠”启动。

## 五、正确卸载

1. 保存尚未发送的内容。
2. 完全退出所有 Codex 窗口。卸载器不会替你强制关闭 Codex。
3. 从以下任一入口运行卸载：
   - 桌面或开始菜单中的“卸载 Codex 琪亚娜桌宠”；
   - Windows `设置 > 应用 > 已安装的应用` 中的“Codex 琪亚娜增强桌宠”；
   - 解压目录中的 `卸载 Codex 琪亚娜增强桌宠.cmd`。
4. 等待出现“卸载完成”提示。
5. 之后直接使用原来的普通 Codex 入口启动即可。

### 如果弹出“请先完全退出 Codex”

这表示安全保护生效，卸载已经中止，当前文件没有被删除。请：

1. 关闭全部 Codex 窗口；
2. 必要时在任务管理器中确认 Codex/ChatGPT 进程已经退出；
3. 再运行一次卸载器。

视频结尾展示的就是这个中止提示，所以不能把那段画面理解为已经完成卸载。

### 卸载时会保留什么

- 如果安装前已有同 ID 的桌宠，卸载器会恢复原文件。
- 如果安装后手动修改过桌宠文件，卸载器会保留它，避免误删。
- Codex 的聊天、项目、登录状态和界面文件不会被删除。

因此，卸载后仍能在 Pets 列表看到同名桌宠，不一定代表卸载失败；它可能是被恢复或因用户修改而保留的版本。

## 六、常见问题

### Pets 列表里没有“时砾逐光”

先在普通 Codex 的 `设置 > Pets（宠物）` 中刷新自定义桌宠。如果仍未出现，完全退出并重新打开一次普通 Codex。

### 桌宠出现了，但不会自动奔跑或跟随鼠标

确认已经完全退出普通 Codex，然后从“Codex 琪亚娜桌宠”快捷方式启动。普通入口不会加载增强逻辑。

### 安装器提示完整主题仍在运行

先使用完整主题自己的恢复或卸载入口关闭主题，再安装纯桌宠版。两套运行状态不要同时启用。

### Windows 显示未知发布者或信誉提示

当前安装器没有商业代码签名。请只从本仓库 Releases 下载，并先核对 SHA-256。确认来源和哈希无误后，再决定是否运行。

### Codex 更新后增强动作失效

先查看 [最新 Release](https://github.com/3073717189/Codex-Kiana-Pet-Enhancer/releases/latest) 是否已有兼容版本。重新安装后仍有问题，请提交脱敏后的错误信息。
